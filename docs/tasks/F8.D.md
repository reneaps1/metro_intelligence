## 🎯 Objetivo (derivado)

Calcular Cp/Cpk (capacidad de proceso) y límites de control I-MR básicos (UCL/LCL vía moving range) para alimentar los gráficos de tendencia/SPC en el dashboard.

## 📋 Contexto

`docs/testing-strategy.md` es explícito: "Cp/Cpk/Pp/Ppk against hand-calculated reference values" y "each Nelson/Western Electric rule with synthetic series that do and don't trigger" — **esto no es opcional para esta tarea, es literalmente parte de la estrategia de testing ya documentada del proyecto.** Para la versión demo, probablemente baste con Cp/Cpk + límites I-MR; las reglas Nelson/Western Electric completas pueden ser un refinamiento posterior si el tiempo no alcanza — pero dilo explícitamente si las omites.

## ✅ Alcance (derivado)

- `backend/app/engines/spc/capability.py`: `cp(values, spec) -> float`, `cpk(values, spec) -> float` — funciones puras. **El mock frontend (`frontend/src/lib/mock/generator.ts`, función `cpk()`) ya tiene una implementación de referencia en TypeScript** que puedes traducir/verificar contra los mismos vectores, aunque el engine real en Python debe tener sus propios tests independientes.
- `backend/app/engines/spc/control_limits.py`: cálculo de UCL/LCL para I-MR (media móvil de rangos, constantes estadísticas estándar — no las inventes, usa las constantes conocidas para subgrupos de tamaño 2, típicas de I-MR).
