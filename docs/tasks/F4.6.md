## 🎯 Objetivo

Exponer lectura de mediciones: listado de runs con filtros (parte, fecha, máquina, programa), detalle de run con samples/results, y serie temporal por característica optimizada para gráficas (la ruta que alimenta trends y SPC en la UI).

## 📋 Contexto

Solo lectura — las mediciones entran por F4.5 y son inmutables (trigger insert-only en `measurement_results`, ver `0003_measurement.py`). El endpoint de serie temporal es el más usado por el frontend: debe soportar rangos de fechas, downsampling opcional, y devolver la spec vigente por punto (recuerda: `catalog_specifications` está versionada — un resultado antiguo puede corresponder a una versión de spec distinta a la actual).

## ✅ Alcance

- `GET /measurement-runs` (paginado, filtros), `GET /measurement-runs/{id}`.
- `GET /characteristics/{id}/series?from&to&max_points` — incluye nominal/límites de la spec vigente **en cada tramo** (no solo la spec actual, si el rango cruza una versión de spec).
- Respuestas tipadas con unidades y estado OK/NOK por punto.
- Verifica los índices con `EXPLAIN` sobre el dataset seed completo — `measurement_results` tiene un índice `(characteristic_id, measured_at)` ya creado en la migración, tu query debe poder usarlo.

## 🚫 Fuera de alcance

Cálculos SPC (eso es F8.D); escritura de mediciones (eso es F4.5).

## Referencias obligatorias

- `backend/app/models/measurement.py`, `backend/alembic/versions/0003_measurement.py` (partición mensual, índice, trigger insert-only).
- **`frontend/src/components/charts/TrendChart.tsx` y `frontend/src/lib/mock/fixtures.ts`** (mock existente, F5.M) — el mock ya define exactamente qué forma de puntos (`{measuredAt, value, deviation, isOk, sampleIndex}`) espera el gráfico. Tu API real debería poder alimentar ese mismo componente sin cambios (ver F5.7).

## ✔️ Criterios de aceptación

- [ ] La serie de 90 días de una característica responde en menos de 300ms sobre el dataset seed completo.
- [ ] La serie refleja cambios de versión de spec en los tramos correctos (test).
- [ ] RBAC aplicado (`viewer` puede leer, no puede escribir nada — no hay endpoints de escritura aquí de todos modos).
