## ⚠️ Importante: la mayoría de estos componentes ya existen y están validados

`frontend/src/components/charts/` (F5.M) ya contiene:

- **`TrendChart.tsx`** — esto ES el I-MR chart pedido: serie + nominal (gris discontinuo) + límites de tolerancia (rojo discontinuo) + violaciones marcadas con triángulo (color+forma, no solo color). Construido con Recharts.
- **`Histogram.tsx`** — capability histogram con líneas de referencia de límites.
- **`RiskHeatmap.tsx`** — características × tiempo, rampa secuencial de un solo hue, valor mostrado como texto en cada celda (no solo color), implementado como una tabla HTML semántica real (accesible, no un canvas).
- **`Sparkline.tsx`** — SVG polyline minimalista.
- **`StatTile.tsx`** (en `components/ui/`, no `components/charts/`) — valor + delta + sparkline.

Este es el proyecto que validó la paleta de charts (`docs/design/design-system.md` §1.4) contra el validador de accesibilidad de la skill `dataviz` — dos colores (`--chart-2` teal, `--chart-6` slate) no pasaron el piso de croma, así que el mock se apoya en leyenda/etiqueta directa como mitigación en vez de depender solo del color. Ten esto en cuenta si tocas la paleta.

## 🎯 Lo que realmente falta

- **`ControlChart.tsx`** con X̄-R (el `TrendChart` actual solo cubre I-MR — X̄-R para datos subagrupados no existe).
- UCL/LCL (límites de control estadísticos, ámbar punteado) — el `TrendChart` actual solo dibuja límites de **tolerancia**, no límites de control SPC calculados. Necesitas los dos tipos de línea de referencia distinguibles simultáneamente una vez F8.D provea los cálculos reales.
- Storybook stories con datos sintéticos (el mock se verificó con Playwright end-to-end, no tiene stories aisladas por componente).
- Documentar la elección de librería (Recharts) en `docs/design/decisions.md` — esto no se hizo formalmente, solo se usó.
- Verificar el criterio de "5k puntos sin jank" — el mock nunca se probó con series tan largas (el dataset de F3.3 tiene ~68 puntos por característica).
