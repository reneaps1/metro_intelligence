# Seed Data Strategy (Demo)

All demo data is **fictitious by construction** — generated by code in `seed/`, never copied from any real customer. CI includes a guard test rejecting real-looking identifiers (CLAUDE.md §20).

## Naming scheme (deliberately artificial)

- Parts: `MI-DEMO-1001` … ; names like "Bracket Front Left (Demo)".
- Plant/lines: `Plant Demo Norte`, lines `L1..L3`, machines `CMM-01`, `CMM-02`, `SCAN-01`.
- Users: `ana.garcia@demo.local` etc. — `.local` domain only.

## Dataset design (built to showcase the engines)

- 3 product families, ~8 part numbers, 10–25 characteristics each (diameters, positions, flatness, profiles) with realistic automotive-style tolerances (±0.05–0.5 mm) and classifications (CC/SC/standard).
- ~90 days of measurement history, shift-patterned frequency.
- Injected patterns per characteristic (parameterized generators):
  - stable & capable (Cpk > 1.67) → candidates for **frequency reduction**,
  - slow drift toward a limit → **trend/risk alerts**,
  - shift after a `ProcessEvent` (tool change) → **event correlation + post-event validation**,
  - high variance near limits (Cpk < 1.0) → **frequency increase / immediate inspection**,
  - occasional NOK outliers → compliance + alert flow.
- Process events: tool changes, maintenance, material lot changes, machine adjustments — timestamped to explain the injected patterns.
- Pre-seeded recommendation/decision history so Decision Memory screens aren't empty.
- Users: one per role (viewer, metrologist, quality_engineer, admin, auditor), demo passwords set at seed time from env — never hardcoded.

## Implementation

- `seed/generators/` (Python, numpy): deterministic with fixed random seed → reproducible demos.
- Output: direct DB insert via backend models + sample CSV/Excel files in `seed/sample_files/` to demo the *import* flow live.
- `make seed` / `python -m seed` resets and repopulates a demo environment.

## Validation

- Seed output passes schema constraints (FKs, unique balloon numbers, spec versions).
- Statistical sanity: generated Cpk values match intended scenario labels (test asserts ranges).
- Confidentiality lint: no identifier matches real OEM part-number patterns; no real plant names.
